# **《如视海外2025年庞加莱线上内购会》邀请函管理系统 - 产品需求文档**

## **一、 项目概述**

### **1.1 项目背景**
为如视海外"2025年庞加莱线上内购会"打造邀请函管理系统，支持销售团队高效创建多语言邀请函，追踪客户反馈。

### **1.2 核心价值**
1. 快速创建专业的多语言邀请函
2. 提供优美的多语言邀请体验
3. 客户接受/拒绝后立即通知销售跟进
4. 提供参与度分析数据

### **1.3 用户角色**
| 角色 | 职责 | 使用语言 |
|------|------|----------|
| **超管** | 系统配置、销售账号管理、模板管理 | 中文界面 |
| **销售** | 创建邀请函、分享链接、跟进客户 | 中文界面 |
| **客户** | 查看邀请函、选择接受或拒绝 | 8种语言 |

### **1.4 目标客户区域**
- **欧美市场**：英语
- **东亚市场**：中文、日语、韩语
- **东南亚市场**：印尼语、泰语、越南语
- **中东市场**：阿拉伯语（RTL）

## **二、 系统核心功能**

### **2.1 超管后台（中文界面）**

#### **2.1.1 销售账号管理**
- **创建销售账号**：
  - 填写销售姓名、销售代号（固定用户名）
  - 填写企业微信ID（wechatId, 用于精准通知）
  - 系统自动生成随机密码
  - 创建成功后显示账号信息供超管分发
- **账号状态控制**：启用/禁用账号
- **密码重置**：重新生成随机密码

#### **2.1.2 模板管理**
- 管理至少10个不同风格的邀请函模板
- 每个模板支持8种语言预览
- 启用/禁用模板控制

#### **2.1.3 系统配置**
- **设置活动时间**：固定北京时间（YYYY-MM-DD HH:MM格式，24小时制）
- 设置内购会参与链接
- 配置企业微信机器人Webhook

### **2.2 销售后台（中文界面）**

#### 2.2.1 登录方式
- **Web表单登录**：使用超管分发的用户名（销售代码）和密码登录，提供更友好的界面和错误提示
- **登录时效**：7天（通过Cookie/Session管理）

#### **2.2.2 创建邀请函**
1. **选择模板**：从10+模板中选择，预览不同语言效果
2. **设置语言**：选择邀请函显示语言（8种可选）
3. **填写客户信息**：
   - 客户姓名（必填）
   - 销售备注（仅自己可见，选填）
4. **生成邀请函**：系统生成唯一访问链接和二维码
5. **预览确认**：可切换语言预览

#### **2.2.3 分享功能**
- **复制链接**：一键复制邀请函链接
- **下载二维码**：生成并下载邀请函二维码

#### **2.2.4 邀请函管理**
- 查看所有创建的邀请函列表
- 查看每个邀请函的状态：未访问/已打开/已接受/已拒绝
- 查看详细数据：访问时间、访问设备

#### **2.2.5 客户反馈追踪**
- 实时显示客户状态变化
- 查看已接受客户的专属邀请码

#### **2.2.6 数据看板**
- 个人邀请函统计
- 接受率/拒绝率分析

### **2.3 客户体验（H5页面）**

#### **2.3.1 多语言展示**
- 根据销售创建时选择的语言显示
- 支持8种语言，包括RTL语言（阿拉伯语）

#### **2.3.2 时间显示方案**
- **统一时间源**：超管后台设置的北京时间
- **客户端格式化**：通过JavaScript API将北京时间格式化为客户本地时间
- **显示格式**：自动适配客户本地日期时间格式

#### **2.3.3 交互流程**
1. **初始状态**：展示邀请函，提供"接受"和"拒绝"按钮
2. **接受邀请**：
   - 显示专属邀请码和内购会参与链接
   - 状态锁定为"已接受"
   - 触发企微通知给销售
3. **拒绝邀请**：
   - 展示遗憾提示文案
   - 提供"重新考虑？"选项
   - 触发企微通知给销售
4. **重新考虑**：从拒绝状态回到初始状态

### **3.4 时间格式化技术方案**

#### **3.4.1 数据存储**
- **服务器存储**：北京时间（UTC+8）的ISO格式字符串
- **示例**："2025-06-15T14:30:00+08:00"

#### **3.4.2 前端格式化**
```javascript
// 从前端API获取北京时间
const beijingTime = "2025-06-15T14:30:00+08:00";

// 转换为本地时间显示
const formatLocalTime = (beijingTime, language) => {
  const date = new Date(beijingTime);
  
  // 根据语言设置格式化选项
  const options = {
    year: 'numeric',
    month: 'long',
    day: 'numeric',
    hour: '2-digit',
    minute: '2-digit',
    timeZoneName: 'short'
  };
  
  // 根据邀请函语言格式化
  return date.toLocaleString(language, options);
};

// 示例输出：
// 英语："June 15, 2025 at 2:30 PM PDT"
// 中文："2025年6月15日 14:30 GMT+8"
// 日语："2025年6月15日 14時30分 JST"
```

#### **3.4.3 多语言时间格式**
- **英语**：Month DD, YYYY at HH:MM AM/PM (Time Zone)
- **中文**：YYYY年MM月DD日 HH:MM (时区)
- **日语**：YYYY年MM月DD日 HH時MM分 (タイムゾーン)
- **韩语**：YYYY년 MM월 DD일 HH시 MM분 (시간대)
- **阿拉伯语**：DD MMMM YYYY، الساعة HH:MM (التوقيت)

## **八、 时间管理方案**

### **8.1 时间设置与存储**
- **超管设置**：固定北京时间（YYYY-MM-DD HH:MM）
- **存储格式**：ISO 8601格式，包含时区信息
- **示例存储**："2025-06-15T14:30:00+08:00"

### **8.2 客户端时间格式化**
- **技术实现**：使用JavaScript的Intl API
- **时区检测**：自动检测客户端时区
- **格式适配**：根据邀请函语言和本地习惯格式化

### **8.3 时间显示规则**
1. **服务器端**：统一存储北京时间
2. **客户端**：自动转换为本地时间显示
3. **时区标注**：显示客户本地时区信息
4. **回退机制**：如无法检测时区，显示UTC时间

### **8.4 API设计**
```javascript
// 邀请函数据API响应示例
{
  "invitation": {
    "id": "inv_001",
    "title": "如视海外2025年庞加莱线上内购会",
    "customer_name": "张三",
    "language": "zh-CN",
    "event_time": "2025-06-15T14:30:00+08:00", // 北京时间
    "sales_note": "重点客户", // 销售备注
    "visit_count": 5, // 访问次数
    "formatted_time": "2025年6月15日 14:30 (北京时间)", 
  }
}

// 客户端格式化逻辑
const beijingTime = response.invitation.event_time;
const language = response.invitation.language;
const localTime = formatLocalTime(beijingTime, language);
```
